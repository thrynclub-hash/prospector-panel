# Phase 02.1 Plan 01: Reusar logo, paleta de cores e fotos do site original Summary

**Redesign generation now scrapes the lead's own site for logo/photos/brand color via one extended Microlink call + a new best-effort HTML scraper, non-blocking, preferred over Places Photos when available.**

## Performance

- **Tasks:** 6 completed
- **Files modified:** 5 modified, 1 created

## Accomplishments
- `RedesignContent.theme?: { primaryColor: string | null } | null` added as a read-only top-level sibling field, matching the `facts` precedent, without touching the frozen shape's existing keys or requiring a data migration
- `lib/screenshot.ts`'s `captureScreenshot` became `captureSiteVisuals`, resolving both the "before" screenshot and the site's logo (favicon/apple-touch-icon chain) in a single Microlink call
- New `lib/site-scrape.ts` (`scrapeSiteAssets`) extracts up to 3 content photos (regex `<img>` + filename/dimension heuristic) and a `theme-color` meta tag from one plain `fetch`, following the exact non-throwing pattern of `lib/email-scrape.ts`
- `generate/route.ts` wires all three functions in, physically positioned after the try/catch that returns 502s, so scraping failures degrade silently to today's behavior (neutral template + Places Photos)
- `redesign-preview.tsx` (shared by the private preview and the public `/demo/[slug]` page) now renders `theme.primaryColor` as an accent color on the hero background and section headings, falling back to the neutral `slate-900` template when absent
