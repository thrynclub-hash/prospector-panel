# Phase 02.2 Plan 01: Data foundation (opening hours + photo cap) Summary

**Places Details now requests `regularOpeningHours` in Portuguese and populates `facts.openingHours` (never invented, `null` when absent); the photo cap used by redesign generation rises from 3 to 8 on both photo sources, symmetrically.**

## Performance

- **Tasks:** 4 completed
- **Files modified:** 4 modified, 0 created

## Accomplishments
- `RedesignContent.facts.openingHours?: string[] | null` added inside `facts` (not a sibling like `theme`) -- it's a verified fact from the business's own Places profile, same source as `phone`/`address`/`rating`
- `DETAILS_FIELD_MASK` (Place Details, used only at redesign-generation time) now includes `regularOpeningHours`; `PRO_TIER_FIELD_MASK` (`searchText`, used at lead search/listing time) is untouched
- `getDetails` now requests `languageCode=pt-BR`, so `weekdayDescriptions` comes back already localized (e.g. "segunda-feira: 09:00 – 18:00")
- `generate/route.ts` populates `facts.openingHours` straight from the same `details` response already fetched (`details.regularOpeningHours?.weekdayDescriptions ?? null`) -- never invented, `null` when the business declares no hours on Maps
- Photo cap raised from 3 to 8 in both fallback sources: Places Photos (`generate/route.ts`) and the site-original scrape (`lib/site-scrape.ts`'s `MAX_PHOTOS`) -- kept symmetric on purpose so the preferred source (site original) is never capped lower than the fallback
